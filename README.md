## 1. Introduction
Open Source Software (OSS) plays a major role in modern computer science. It allows users to study, modify, and distribute code freely.

This report analyzes Git, a distributed version control system, including its origin, licensing, ethical aspects, and role in the open-source ecosystem.

## Part A — Origin and Philosophy

### A1. Origin of Git
Git was created in 2005 by Linus Torvalds after the Linux community lost access to BitKeeper.

Key features:
- Distributed system  
- High speed and performance  
- Reliable for large projects  

---

### A2. License Analysis

#### Software License
A software license defines how software can be used, modified, and shared.

#### Four Freedoms
- Freedom to run the program  
- Freedom to study and modify  
- Freedom to redistribute  
- Freedom to distribute modified versions  

#### GPL v2
- Uses **copyleft**  
- Ensures all modifications remain open source  

#### GPL vs MIT
- GPL → strict, requires openness  
- MIT → permissive, allows proprietary use  

---

### A3. Ethics of Open Source
- Encourages collaboration and innovation  
- Helps students and developers  
- Businesses need profit → may keep software closed  
- Security concerns in sensitive systems  

Conclusion: Not all software should be open source, but it is highly beneficial.
